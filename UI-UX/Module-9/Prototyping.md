
## Learning Objectives
- Identify and apply appropriate prototyping triggers to match specific interaction goals  
- Demonstrate the ability to chain multiple triggers and actions to create realistic user flows  
- Implement responsive design changes that adapt layouts, navigation, and content for different screen sizes  
- Differentiate between decorative animations and functional animations that guide the user  
- Optimize interactions for both performance and accessibility  

---

## Activity Outlines

### 1. Authentication Screen

#### a. Input Interaction
- Input fields should accept user input in real time  
- Use appropriate triggers for:
  - Typing via keyboard  
  - Clicking/tapping input fields  

#### b. Conditional Navigation
- Implement logic to:
  - Navigate to the main screen if credentials are correct  
  - Display an error message if credentials are incorrect  
  - Prompt the user to try again  

---

### 2. Quantity Selector (Add/Remove Items)
- The **plus (+)** button should increase item quantity  
- The **minus (–)** button should decrease item quantity  
- Ensure interaction works on both click and tap  

---

### 3. Progress Button Loader
- When the user clicks the button to proceed:
  - Show a loading indicator (loader)  
  - Then transition to the next step  

---

### 4. Order Confirmation Feedback
- After placing an order successfully:
  - The button should display a loading state  
  - Then change to a **success state** (e.g., turn green)
