# Angular Fundamentals: Dessert Shop App

## Learning Objectives

By the end of this lab, learners should be able to:

- Set up and configure an Angular project using the Angular CLI.
- Create and organize reusable components for better maintainability.
- Implement various forms of data binding to connect logic and templates.
- Use structural and attribute directives to control and style DOM elements dynamically.
- Manage communication between parent and child components.
- Display and update dynamic data (like cart items) within the application.
- Deploy an Angular application to a hosting platform.

## Introduction

In this lab, learners will apply fundamental Angular concepts to develop a **Dessert Shop App**, showcasing the use of components, templates, data binding, and directives to create a dynamic and interactive web application.

This lab emphasizes understanding the structure of Angular applications, creating reusable building blocks, and implementing responsive UI behaviors using Angular's template syntax.

### Lab Requirements

- Node.js (v18 or later)
- Angular CLI (latest version)
- Code Editor (VS Code recommended)
- GitHub account
- Dessert Shop design assets
- Netlify or Vercel account for deployment

**Estimated Time:** 4 hours


## Project Setup
### Task 1:
Create and configure a new Angular project for the Dessert Shop App.

Ensure that the initial application runs successfully in the browser.

## Tasks

### Task 2: Create Core Components

Design and organize the main building blocks of the application.

The app should include separate components for displaying available products, individual product details, and the shopping cart.

### Task 3: Implement Data Binding

Connect data between your templates and component logic.

Ensure product details such as name, image, and price are dynamically displayed, and user interactions (like adding items to the cart) update the data accordingly.

### Task 4: Use Directives for Dynamic Content

Apply Angular's structural and attribute directives to make your UI responsive to data changes.

Render lists of products, display messages when no products are available, and visually distinguish selected or highlighted items.

### Task 5: Manage Component Communication

Enable interaction between components.

Ensure that user actions in product components (like adding an item) update the shared cart view in another component.

Keep shared data organized within parent components.

### Task 6: Display and Update the Cart

Create a section that dynamically displays all selected items.

Ensure the total amount updates correctly and that users can clear or modify their selections.

### Task 7: Styling and Layout

Apply consistent styling to create a user-friendly interface.

Ensure the design is responsive and closely matches the provided Dessert Shop layout reference.

### Task 8: Deployment

Prepare the project for production and deploy it to a live hosting platform.

Share the live application link along with your GitHub repository.

### Bonus Task (Optional): Enhancements

- Display a message when an item is successfully added to the cart.
- Visually highlight products already in the cart.
- Format currency values appropriately for your region.
